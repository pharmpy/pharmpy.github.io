Selection criteria#

Pharmpy tools support multiple different selection criteria, and most tools allow the user to specify which to use.

Objective function value (OFV)#

Uses the objective function value as a selection critera, most often \(-2\text{log}\mathcal{L}(\hat{\theta} \mid y)\).

Likelihood ratio test (LRT)#

Performs a likelihood ratio test, where \(\alpha\) is the cutoff p-value.

Note

In tools that use likelihood ratio test (e.g. COVSearch and RUVSearch) the models are always nested, and thus LRT is applicable. However, if this is set for some of the other tools this might not be the case, but currently Pharmpy does not detect whether models are nested. Thus, if using this in another tool, LRT might not be applicable for all models in all tools.

Aikaike information criterion (AIC)#

The Aikaike information criterion can also be calculated. In order to calculate the AIC in Pharmpy standalone, you can use the pharmpy.modeling.calculate_aic()-function. The AIC is described as follows:

\[AIC = -2\text{log}\mathcal{L}(\hat{\theta} \mid y) + 2dim({\theta})\]

Bayesian information criterion (BIC)#

Pharmpy supports multiple types of BIC. In order to calculate the BIC in Pharmpy standalone, you can use the pharmpy.modeling.calculate_bic()-function.

Fixed#

For linear models, the BIC [1] is described as follows:

\[BIC_{fixed} = -2\text{log}\mathcal{L}(\hat{\theta} \mid y) + dim(\theta)\text{log} n_{tot}\]

Where \(\theta\) is estimated parameters and \(n_{tot}\) is the number of observations.

Mixed#

For mixed effect models, BIC takes the number of random vs. fixed parameters into consideration [2], and is described as follows:

\[BIC_{mixed} = -2\text{log}\mathcal{L}(\hat{\theta} \mid y) + dim({\theta_R})\text{log} N + dim({\theta_F})\text{log} n_{tot}\]

Where \(\theta_R\) is random parameters, \(N\) is the number of subjects, \(\theta_F\) is fixed parameters, and \(n_{tot}\) is the number of observations.

IIV#

The BIC for IIV [3] is described as follows:

\[BIC_{IIV} = -2\text{log}\mathcal{L}(\hat{\theta} \mid y) + dim({\omega_{IIV}})\text{log} N\]

Where \(\omega_{IIV}\) is IIV parameters and \(N\) is the number of subjects.

Random#

The BIC for random effects is described as follows:

\[BIC_{random} = -2\text{log}\mathcal{L}(\hat{\theta} \mid y) + dim({\theta})\text{log} N\]

Where \(\theta\) is parameters and \(N\) is the number of subjects.

Modified BIC (mBIC)#

The mBIC selection criterion [4] is an additional penalty term, and is combined with any of the above mentioned types of BIC. mBIC was introduced to address the problem of multiple testing, and penalizes against the size of the search space and the complexity of the candidate model.

Currently, mBIC supports evaluation of the structural PK model and the IIV model, and there are two different penalty calculations for each of them. This means that when e.g. evaluation the structural model, the ModelSearch tool only uses mBIC for the structural model.

Note

mBIC is currently only supported by ModelSearch, IIVSearch, and IOVSearch.

mBIC for the structural model#

In mBIC for the structural model, different components of the model structure penalizes differently (see table below). For the structural model, we can define \(mBIC_{structural}\) with the following equation:

\[mBIC_{structural} = BIC_{mixed} + 2 k_M \text{log}\frac{M}{E}\]

Where \(k_M\) is the number of structural predictors in the candidate model, \(M\) is the number of structural predictors in the search space, and \(E\) is the a priori expected number of predictors. The \(E\)-value is set by the user, and is bounded by \(0 < E < M\) (where 0 is no expected number of predictors and thus infinite penalty, and \(M\) is all predictors in the search space which means no penalty).

We can view this as four different categories:

  • Mutually exclusive: one of options have to be selected

    • FO/ZO absorption, FO/MM elimination

    • Contribution to \(M\) and \(k_M\) are each 0

  • On/off: whether a component is active or not

    • Lagtime or no lag time

    • Contribution to \(M\) and \(k_M\) are each 0 when “off”, 1 when “on”

  • Stackable: combinations that can be stacked

    • Peripherals, going from FO/ZO absorption to SEQ-ZO-FO, going from FO/MM elimination to MIX-FO-MM

    • Contribution to \(M\) and \(k_M\) are both increased increased by the number of options (e.g. having 0-2 peripherals in the search space will give \(M=2\) and \(k_M\) of how many peripherals is in candidate)

  • Non-stackable: numeric option which is mutually exclusive

    • Transits

    • Contribution to \(M\) is the number of non \(k_M\) are both increased increased by the number of non-stackable options (e.g. having 0-2 transits in the search space will give \(M=2\) and \(k_M\) of 1 in any candidate with transits)

mBIC for the IIV model#

In mBIC for the IIV model, the penalty consists of one part for the diagonal omegas and one part for the off-diagonals. For the IIV model, we can define \(mBIC_{IIV}\) with the following equation:

\[mBIC_{IIV} = BIC_{IIV} + 2 k_p \text{log}\frac{p}{E_p} + 2 k_q \text{log}\frac{q}{E_q}\]

Where \(k_p\) and \(k_q\) are the number of diagonal and off-diagonal omega predictors in the candidate model, \(p\) and \(q\) are the number of diagonal and off-diagonal omega predictors in the search space, and \(E_p\) and \(E_q\) is the a priori expected number of diagonal and off-diagonal omega predictors.

If you’re only testing the number of diagonal omegas, the penalty related to the off-diagonal omegas is omitted.
