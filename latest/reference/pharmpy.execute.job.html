pharmpy.execute.job module¶

Executed Job¶

A job unit, for non-blocking execution and communication (of subprocesses).

Can be used raw for running subprocesses asynchronously.

Definitions¶

class pharmpy.execute.job.Job(command, cwd=None, stdout=None, stderr=None, callback=None, keepends=False)[source]¶

Bases: threading.Thread

A job of an Engine running in a RunDirectory (via an Environment).

Responsible for generating and wrapping a subprocess for execution and monitoring. Inherits from Thread to run in separate thread (non-blocking monitoring). Call start() to execute (run() considered low-level API).

Parameters

  • command – Program and arguments (iterable).

  • cwd – Working directory.

  • stdout – Callback, for each line of stdout.

  • stderr – Callback, for each line of stderr.

  • callback – Callback, for when task completes.

  • keepends – Keep line breaks for individual lines?

Note

Callback functions (arguments stdout, stderr and callback) must be threadsafe.

async wait(timeout=None, poll=1)[source]¶

Awaits job completion.

Parameters

  • timeout – Maximum wait time in seconds.

  • poll – Wait between each status check of job.

Raises

TimeoutError – Job did not complete in timeout seconds.

Use None for timeout to wait forever. Polling impossible to avoid with Python’s GIL (don’t event try)!
