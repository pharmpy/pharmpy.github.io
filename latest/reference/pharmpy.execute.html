pharmpy.execute package¶

Model Execution¶

Central support for executing models for evaluation, estimation and simulation tasks. Useful for:

  • Model objects to, for example, evaluate themselves on request

  • Full workflows in implementations of Tool (e.g. FREM)

This package defines four classes:

Job

A job unit. Contains an asynchronous process and exposes blocking and non-blocking stdout.

RunDirectory

Run directory with cleaning and safe unlink operations. Invoking directory, where are the models? Which files to copy where?

Environment

Platform (e.g. Linux) & system (e.g. SLURM) implementation. The cluster/local or OS etc. to start jobs on.

Engine

Creates Job objects, executing some task, for a specific implementation. Contains Environment object. The focal point for implementation inheritance.

Where Engine is the critical unit for a implementation to inherit, e.g. NONMEM7.

Environment require inheritance for a specific purpose (e.g. Linux, SLURM, dummy directory, etc.), but not an implementation. RunDirectory and Job will likely remain model (type) agnostic.
